Analyse pannes pluviomètre

Electronique - analyse de pannes
Electronicien Science à l'Institut Polaire Français • 2022

J'ai diagnostiqué les pannes d'un pluviomètre dont l'alimentation avait endomagé plusieurs organes à différents niveaux.

La communication USB ne renvoyait aucune donnée sur l'interface graphique propriétaire. J'ai donc interrogé l'instrument avec le protocole RS-485.


Les données n'étant toujours pas monitorées j'ai interrogé l'instrument avec le protocole i2c. J'ai programmé un Arduino pour discuter avec l'instrument via ce protocole. Un oscilloscope m'a permis de m'assurer de la bonne transmission des trames.


Afin de pousser les test le plus loin possible j'ai déplacé l'instrument dans le labo d'électronqiue.


La carte électronique ne présentait aucun dommage visible à l'euil nu. J'ai donc essayé tous les protocoles de communiction possible avant de déclarer l'instrument définitivement en panne.
